# Parser 재확인 요청 — 마스터테이블 정합성 검증 의심 fail

## 배경

Validation 측에서 마스터테이블 자기완결 정합성 검증(`scripts/validate_master_tables.py`)을 돌려 아래 의심 fail을 발견했다. 각 항목의 **DART raw를 다시 확인**하고 마스터테이블 값을 정정 요청.

**대상 파일**
- `data/dart/viz/pl_breakdown_master.json` — P&L breakdown (백만원, 32사 × 13분기, long-format)
- `CSM_waterfall.json` — CSM waterfall 6-step (억원, 23사 × 13분기, long-format)

**전체 의심 fail 목록(101건, 상세 raw 포함)**: `data/_derived/master_validation_fails.json`

**검증 룰**
1. **PL_BRIDGE** (pl_breakdown_master 내부, 백만원): 8단 등식 — `생명장기원수손익 = 원수CSM상각+원수RA+원수예실차+기타원수` → `생명장기손익 = 원수+재보험` → `보험손익 = 생명장기손익+자동차+일반+기타영업수익−기타사업비용` → `영업이익 = 보험손익+투자손익` → `세전 = 영업이익+영업외손익` → `당기순이익 = 세전−법인세`
2. **CLOSING_IDENTITY** (CSM_waterfall 내부, 억원): `기초+신계약+이자+가정및경험조정+CSM상각 = 기말`
3. **CSM_CROSSCHECK** (cross-table, 연말 기준): `pl.원수CSM상각(백만, +) + wf.CSM상각(억×100, −) ≈ 0`

tolerance: 각 등식 `max(0.1%·|기준값|, 200백만 또는 2억)`.

---

## 그룹 A — PL_breakdown 기타영업수익/기타사업비용 (52건, 최우선)

등식 `보험손익 = 생명장기손익 + 자동차 + 일반 + 기타영업수익 − 기타사업비용`이 안 닫힘. 두 하위 원인:

**A1. 기타영업수익이 0으로 누락** (보험손익과의 차이가 기타사업비용과 일치)
- KB손해 2023.1Q: 보험손익 258,881 vs 합 166,316 (diff −92,565 = 기타사업비용). 기타영업수익=0.
- DB손해 2023.1Q: 보험손익 431,045, 기타영업수익=0, diff −34,809.
- 대상: **KB손해 / DB손해 / 현대해상 / 흥국화재 / 한화손해 / 메리츠** (대부분 2023 전분기 + 일부 2024~2025)

**A2. 기타사업비용 부호 오류 (음수로 들어감 — 비용은 양수여야)**
- 농협생명 2024.1Q: 기타사업비용 = **−10,619** (diff +21,239)
- 삼성생명 2023.3Q: 기타사업비용 = **−36,766** (diff +73,532)

**확인 요청**: 연결 포괄손익계산서의 `기타영업수익` / `기타사업비용` 행을 분기별로 재추출. 부호 통일(비용=양수). 누락 분기 채움.

---

## 그룹 B — PL_breakdown 투자손익 과대 (영업이익 등식 11건)

`영업이익 = 보험손익 + 투자손익`에서 우변이 영업이익보다 크게 초과 → **투자손익 과대추출** 의심 (누적/단일분기 혼재 또는 중복합산).
- KB라이프 2023.3Q: 투자손익 953,978 → diff +764,100 (거대)
- 동양생명 2023.1Q: 투자손익 440,261 → diff +305,234
- DB생명 2023.1Q: 투자손익 156,135 → diff +96,368
- 대상: **DB생명 / KB라이프 / 동양생명 / 신한라이프 / 케이디비생명** (주로 2023 초기분기)

**확인 요청**: `투자손익 = 투자이익 + 보험금융손익` 추출 시 YTD 누적/단일분기 혼재 여부, 중복합산 여부 점검.

---

## 그룹 C — CSM_waterfall closing identity 안 닫힘 (40건)

`기초+신계약+이자+가정및경험조정+CSM상각 ≠ 기말`. step 중 하나 추출 오류.
- 대상(건수): **에이비엘생명(9) / 케이디비생명(6) / 미래에셋(3) / 케이비라이프(3) / 흥국화재(3) / 한화생명(2)** + 산발
- 예: 미래에셋 2024.3Q — 기말 21,689 vs 합 20,389 (diff −6.0%)
- 큰 건: 흥국화재 2023.3Q −22%, 에이비엘 2023.3Q +18%, 케이디비 2024.1Q +17%

**확인 요청**: 측정요소별 변동표에서 step별 CSM 칼럼(D/E/F 합) 재추출. 특히 **가정및경험조정**은 잔차가 아니라 독립 추출돼야 함 — 이 값이 어긋나면 다른 step 누락 가능.

## 그룹 D — 원수 CSM상각 pl↔wf 불일치 (4Q 연말, 5%+)

같은 회사·분기 연말 기준 `pl.원수CSM상각` ≠ `wf.CSM상각`.
- 농협생명 2025.4Q: pl 401,662 vs wf 303,530 (+24%)
- NH농협손해 2025.4Q: pl 199,550 vs wf 241,910 (−21%) / 2023.4Q +19%
- 메리츠 2023.4Q: pl 1,069,760 vs wf 897,770 (+16%)
- 교보생명 2025.4Q: +15% / 케이디비 2025.4Q: +13% / 한화생명 2025.4Q: +8%

**확인 요청**: 두 출처가 같은 **발행 보험계약(원수)** 기준인지 — (a) P&L 보험수익 구성의 "서비스 이전으로 인식한 CSM"(원수CSM상각) vs (b) CSM 변동표의 "당기손익 인식 CSM"(wf CSM상각). 한쪽 추출 오류 또는 범위(원수/재보험) 혼입 점검.

---

## 회신 형식

각 fail에 대해: **재추출값 + 원인 1줄**. 수정 후 마스터테이블(`pl_breakdown_master.json` / `CSM_waterfall.json`) 갱신하면 validation이 재검증:

```
python scripts/validate_master_tables.py
```

회귀 목표: PL_BRIDGE 보험손익/영업이익 fail 해소, CLOSING_IDENTITY 큰 fail(>2%) 해소, CSM_CROSSCHECK 4Q fail(>5%) 해소.
