## 🚫 K-ICS.html 표시 정합 버그 — "적용후" 모드에서 부모후 + 자식전 base 혼합 노출

지금처럼 남겨두면 안 된다."**

### 증상
경과조치 토글을 **적용후**로 두면, 부모 위험액은 적용후 값(예 흥국화재 2026.1Q 생명장기위험액 **16,713**)인데
그 하위위험액은 **적용전 값**(사망 1,112·장해질병 16,026·**해지 8,534**·사업비 3,812·대재해 373)이 그대로
표시됨. 이 조합은 mmult(√(V'·R7·V))가 **안 닫힘**(자식전으로 계산하면 21,398 ≠ 부모후 16,713). 즉 화면에
서로 안 맞는 숫자가 버젓이 조회됨.

### 근본 원인 (K-ICS.html `getRowValue`, 라인 248-254)
```js
function getRowValue(row) {
  const mode = getTransitionMode();
  if (mode === '적용후' && row['값_적용후'] != null) return row['값_적용후'];
  return row['값'];   // ← 적용후 모드인데 값_적용후 결측이면 적용전(값)으로 조용히 폴백

```
`값_적용후`가 null인 행(적용후 세부 미추출/미공시)에서 **적용전 값으로 무조건 폴백** → 부모후 + 자식전
base 혼합. 결측을 "그럴듯한 적용전 숫자"로 가려서 정합 붕괴를 은폐함.

### 요구 (둘 중 하나로 정합 보장)
1. **적용후 모드에서 `값_적용후`가 null이면 적용전으로 폴백하지 말 것.** 표시 안 함(빈칸/"−"/N/A)으로 두어
   "적용후 미제공"을 드러내라. (owner: "미공시면 숫자 안 보이게".)
