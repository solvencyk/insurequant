# \# Agent: Miscellaneous Disclosure (IR \& Bonds)

# 

# \*\*목표:\*\* 원보험사별 자체 IR 홈페이지 크롤링 및 주요 지표, 채권 발행 이력 수집.

# 

# \## 작업 원칙

# \- \*\*후순위 파이프라인:\*\* 이 작업은 K-ICS 및 IFRS17 메인 파이프라인에 영향을 주지 않도록 독립적으로 구성할 것.

# \- \*\*방어적 크롤링 (Defensive Crawling):\*\* IR 게시판은 DOM 구조 개편이 잦음. DOM 파싱 로직과 데이터 정제 로직을 분리할 것.

# \- \*\*Graceful Failure:\*\* A 회사의 IR 페이지 구조 변경으로 에러가 발생해도, 스크립트 전체가 뻗지 않고 에러 로깅 후 B 회사 작업으로 정상 진행되도록 예외 처리(Try-Except)를 꼼꼼히 감쌀 것.
