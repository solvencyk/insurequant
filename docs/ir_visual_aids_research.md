## 조사 목적

`insurequant.html` 같은 다운스트림 시각화에서 어떤 차트·KPI를 IFRS17 데이터 기반으로
보여줘야 하는지 결정하기 위해, 국내 대형 손보·생보사의 최근 IR 자료에서 실제 사용되는
visual aid를 카탈로그화. 사용자가 가설로 든 두 차트(CSM 변동 waterfall / 신계약 CSM
배수 꺾은선)가 업계 표준인지 검증.

## 사용자 아이디어 검증

### ✅ CSM 변동 waterfall — **표준 채택 권장**

조사한 6社 **모두** CSM 변동을 시각화. 6社 중 4社는 waterfall (or waterfall-style
누적 막대). 2社(DB·삼화 3Q)는 표 형식이지만 같은 항목 구조(기초→신계약→이자부리→
조정→상각→기말). 사용자 가설 정확.

**디자인 패턴 합의:**
- 항목: `기초 → 신계약 → 이자부리(unwind) → CSM 조정(가정·경험) → 상각 → 기말` (5-7 단계)
- 색상: 증가(+)·감소(-) 색 구분, 회사별로 상이 (한화생명 적/녹, 삼생 회색계열)
- 추가 정보: 연간 상각률 (%) 텍스트 표시 (DB 9.7%, 삼화 10.9%, 한생 ~9%)
- 비교 단위: 분기 단위 + YTD 누계 둘 다 표시 (삼생·현대해상·삼화 3Q)

### ✅ 신계약 CSM 배수 꺾은선 — **표준 채택 권장**

**용어 통일 필요:** 회사별 표기가 다름.
- 손보 (DB·삼화·현대해상): "**신계약 CSM 배수**" — `신계약 CSM ÷ 초회 (월납환산) 보험료`
- 생보 (삼생·한생): "**신계약 CSM 수익성**" — `신계약 CSM ÷ 월납월초`
- 의미는 동일. 단위 "배 (×)"

사용자가 "신계약 매출 = 수입보험료/원수보험료"라고 가설한 것은 **부정확**. 실제 IR에서 분모는
**월납월초 보험료(=월납환산 신계약보험료)**이며, 이는 **연환산 신계약 매출 ≠ 초년도 수입보험료** 와도 다른 표준화 지표.

**중요:** 분모(월납월초)는 IR 발표자료 본문에 **이미 명시**되어 있어 보험개발원 통계
**별도 크롤링 불필요**. IFRS17 공시(DART §14(6) 신계약 영향)에는 월납월초가 없고
**대신 "최초 인식 시 미래 현금 유출/유입"** 등으로 표기되므로 직접 사용 불가. 따라서:

1. **단기 권장:** IR PDF에서 신계약 CSM 배수를 회사별로 직접 스크래핑 (PDF parsing)
2. **중기 권장:** DART §14(6) `new_business_impact`에서 신계약 CSM 캡처 + 보험개발원
   `월납환산 신계약보험료` 크롤링 결합

**디자인 패턴 합의:**
- 차트: 꺾은선 (분기별 5-point) 또는 막대 + 꺾은선 hybrid
- Series 분리: 보장성/저축성(or 인보험/물보험/저축) 별도 라인
- 단위: 배 (×) — 라벨 직접 표시
- 비교: 전년 동기, 직전 분기
