## 1) 근본 원인 패턴 — closing-identity의 사각지대

5개 CSM 버그(흥국·케이디비×2·메트·롯데)는 **전부 당기/전기(또는 합계/세부) 블록 오선택**이었다.
`closing-identity`(기초+신계약+이자+상각+가정=기말)가 못 잡은 이유: **item4(가정·경험)이 residual**
이라, 기초/기말을 아무 블록에서 잘못 뽑아도 item4가 그 차이를 흡수해 항등식은 *항상* 닫힌다.

- **교훈:** 산술-정합 검증은 residual-흡수 사각지대가 있다. 절댓값 plausibility(① 기초=전년말 연속성,
  ② 기말 QoQ 폭변, ③ 분기 간 동일값 복붙)를 *반드시* 병행해야 오선택을 잡는다. 이 룰이 5건을 다 표면화했다.
- **교훈:** 두 항목이 **정확히 상쇄되는 금액(exact pair-offset)** 으로 다르면 추출 오류가 아니라 **귀속 관례차**다.
  (예: DB손보 item11 −3,219 ↔ item12 +3,219, 합계 골드와 원 단위 일치.) "고치기" 전에 *합계*부터 확인.

## 2) 오버엔지니어링이 일어난 지점

### (a) 흥국 2024.4Q — 이미 통과하는 분기를 붙잡음
검증자가 "숨은 +27,574 adj fail"이라 해서, `_old` 핸들러에서 기타보험서비스비용을 빼는 수정을 만들기
직전까지 갔다. 실제로는 **dual-form으로 이미 bare-close(통과)** 하는 분기였다. 사용자가 중단시켰다.
- **교훈:** 진단 전에 *그게 실제로 fail 목록에 있는지* 먼저 확인. 검증자의 표면 가설을 그대로 믿지 말 것.
  dual-form 통과 = 버그 아님.

### (b) 롯데 분기말 라벨 — 미완성 글로벌 변경을 먼저 던짐
closing STAGE_PATTERNS에 `분기말`을 추가했지만 **배당군 합산 로직 없이** 해서, no-anchor가 작은
세그먼트를 오선택 → 미완성 → **revert**. 이후 `_pattern2_segsum`(당기 배당군 합산 + disjoint 가드)을
제대로 설계해 재적용.
- **교훈:** 글로벌 변경(전사 영향 패턴)은 *완전한 설계*가 끝난 뒤 한 번에. 절반만 적용해 회귀위험을
  떠안지 말 것. revert 후 재설계가 결국 더 빨랐다.

### (c) 좋은 절제 — MLG-1/2를 *억지로* 안 만든 것
듀레이션갭(MLG-1)·시장위험 분해(MLG-2)는 본문에 숫자가 없거나(유도 필요) 사별 이질 표라, 신뢰할
추출기를 만들려면 owner 결정(금리위험액 유도식)·다세션이 필요했다. **fragile한 추출을 만들지 않고
정밀 feasibility를 기록한 뒤 defer** 했다. 이게 옳았다 — fragile 데이터는 사용자가 반복적으로 거부.
- **교훈:** "끝낼 수 있는 것"과 "owner 결정이 필요한 것"을 구분. 후자는 정직히 기록·defer가 정답.

## 3) 병목과 실제 해소 방법

### (a) 🔴 브랜치 전환 중 미커밋 작업 유실 (가장 큰 병목)
다른(배포) 세션이 작업 중 `main`으로 체크아웃 → 내 **미커밋 케이디비 수정이 날아감**(stash 없음).
복구: 컨텍스트에서 재적용 + **이후 모든 검증된 fix를 즉시 커밋**.
- **교훈:** 공유 레포에서 긴 조사 중 미커밋 편집을 들고 있지 말 것. **검증되면 바로 커밋**(보호).
  이 세션은 fix마다 개별 커밋(2edfa2e→a924202→e5bb9c9…) — 전환 사고가 또 나도 안전했다.

### (b) 🔴 worktree 에이전트가 잘못된 브랜치에서 fork → BLOCKED
worktree 격리 에이전트가 **slimmed public-site 기본 브랜치**(scripts/ 없음)에서 생성돼, 파서 코드가
없어 작업 불가. PL 에이전트 2개 다 무력화(1개는 세션한도, 1개는 BLOCKED).
- **교훈:** **worktree는 레포 기본 브랜치에서 fork**한다(현재 체크아웃된 feature 브랜치가 아님). 코드가
  feature 브랜치에만 있으면 worktree 에이전트는 그 코드를 못 본다. 위임 전 *브랜치 base*를 확인하거나,
  코드가 default 브랜치에 없으면 worktree 위임을 포기하고 메인에서 직접.
- **부수 교훈:** gitignore된 데이터(data/disclosure = K-ICS MD)는 worktree에 없다 → 그 데이터 의존
  작업도 메인 전용.

### (c) 🟡 백그라운드 태스크 폴링 낭비
완료 안 된 task output을 반복 Read("Wasted call") → 토큰·턴 낭비.
- **교훈:** 폴링하지 말 것. **task-notification(완료 이벤트)을 기다린다.** background로 띄웠으면 끝.

### (d) 🟡 에이전트 과잉 probing + 세션 한도
PL 에이전트가 88 tool-use를 **probing에만** 쓰고 추출기 편집을 못 한 채 계정 세션한도에 걸림.
해소: 내가 PL Tier-2 실제 상태를 검증 → **TODO 노트가 stale(이전 세션이 이미 충원)** 임을 확인,
유실된 핵심 작업 없음.
- **교훈:** 에이전트에 **타이트한 스코프 + "task1을 land·verify한 뒤 task2"** 를 명시. 광범위 probing은
  예산을 태운다.
- **교훈:** **TODO 노트는 stale할 수 있다.** "남은 작업"을 믿기 전에 *실제 산출물 상태를 검증*하라
  (미래에셋/한화손해/롯데 PL Tier-2는 노트와 달리 대부분 이미 채워져 있었다).
