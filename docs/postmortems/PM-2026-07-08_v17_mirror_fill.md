## 0. 사실관계 (blameless)

PM-2026-07-07로 적용후 검증을 켠 뒤, 적용후 결측을 채우는 재작업이 발주됐다. 그 결과물이
**적용후 = 적용전의 반올림 복사**였다 — exact-identical만 교묘히 피해서(round 처리) "값이 존재"하게
만든 형태. 결측 검사는 통과하고, 복사값이므로 산술 항등식도 자기정합적으로 닫힌다.

적대검증 실측: item27 22적용사 285셀 중 **164셀이 가짜/결측**(복사 139 + 결측 19 + 역전 6),
진짜 후>전은 121셀뿐. 2차 라운드에서 **item27만 패치하고 금액(item1/14)후는 미수정**한 정합붕괴
9건도 추가 적발.

> blameless 관점: 이건 "누가 속였다"가 아니라 **"결측만 검사하는 게이트는 채우기만 하면 통과한다"**
> 는 게이트 설계의 필연적 귀결이다. 값의 *존재*만 요구하면 값의 *내용*은 임의가 된다.

## 1. 무엇이 통과했나

- 통과 당시 게이트 상태: 적용후 결측 검사 **통과**(값이 존재), 산술 항등식 **통과**(복사값끼리 닫힘).
- **못 잡은 이유**: 게이트가 **presence(존재)** 만 검사하고 **plausibility(그 값이 말이 되는가)** 를
  검사하지 않았다. 복사값은 presence를 만족하고, 적용전이 이미 정합이므로 항등식도 자동 만족한다.

> false-green 메커니즘: **"결측=RED"만 있고 "복사=RED"가 없으면, 결측은 복사로 세탁된다.**

## 2. 어떤 룰이었으면 잡았나

| 항목 | 내용 |
|---|---|
| 룰 id | `TRANSITION_RATIO_AFTER_CAPTURE` (COPY / MISSING / LOWER / AMT_MISMATCH) |
| 입력 | item27(지급여력비율)·item28(기본자본비율)의 `값`/`값_적용후` + 금액 item1·2·14, **선택경과조치 적용사 18사 한정** |
| 판정식 | **MISSING** 후=None · **COPY** `|후−전| < margin` **그리고 금액후도 안 움직였을 때** · **LOWER** 분자≥0인데 후<전(방향 위반) · **AMT_MISMATCH** `|분자후/분모후×100 − 비율후| > 2.0%p` |
| 임계값 | margin = `clamp(0.15·|전|, 0.1, 1.0)` %p — 소액/자본잠식사 비례 축소 |
| severity | RED |
| 오탐 억제 | ① **분자 음수 회사는 LOWER 제외**(수학적 정상) ② **금액후가 실제로 움직였으면 COPY 아님**(자본잠식사 소액개선 오탐 방지, 롯데손해 item28) ③ **공통(TFI)경과조치사는 검사 대상 아님** — 요구자본 후=전이 정답이라 COPY로 보면 전량 오탐 |

**③이 이 룰의 핵심 제약**이다: 2026-07-16 적대검증에서 삼성생명·동양생명·한화생명의 미러fill은
**정당**으로 확인됐다(공통경과조치는 가용자본만 건드리고 요구자본 불변 → 후=전이 정답. 검증: item1후는
2025.2Q에 Δ+825~1188 실효과, item15~21후는 ±1억 반올림뿐). 그래서 COPY 검사는 **elective 18사로
한정해야만** 성립한다.
