# Disclosure Validation Tool

원보험사별 disclosure.xlsx 파일들의 정합성을 체크하는 도구입니다.

## 기능

이 도구는 다음 7가지 정합성 규칙을 검증합니다:

### a. 지급여력금액 검증
- **규칙**: `가. 지급여력금액 = 기본자본 + 보완자본`
- **허용오차**: 1 이하

### b. 건전성감독기준 재무상태표 상의 순자산 검증
- **규칙**: `Ⅰ. 건전성감독기준 재무상태표 상의 순자산 = 1.보통주 + 2.자본항목중보통주이외의자본증권 + 3.이익잉여금 + 4.자본조정 + 5.기타포괄손익누계액 + 6.비지배지분 + 7.조정준비금`
- **허용오차**: 1 이하
- **참고**: 비지배지분, 조정준비금은 회사에 따라 없을 수 있음

### c. 보완자본으로 재분류하는 항목 검증
- **규칙**: `Ⅲ. 보완자본으로 재분류하는 항목 ≤ 보완자본`

### d. 분산효과 검증
- **규칙**: `- 분산효과 : (1+2+3+4+5) - Ⅰ = 1.생명장기손해보험위험액 + 2.일반손해보험위험액 + 3.시장위험액 + 4.신용위험액 + 5.운영위험액 - Ⅰ.기본요구자본`
- **허용오차**: 1 이하

### e. 지급여력기준금액 검증
- **규칙**: `나. 지급여력기준금액 (Ⅰ-Ⅱ+Ⅲ) = Ⅰ.기본요구자본 - Ⅱ.법인세조정액 + Ⅲ.기타요구자본(1+2+3)`
- **허용오차**: 1 이하

### f. 기타요구자본 검증
- **규칙**: `Ⅲ. 기타요구자본(1+2+3) = 1.업권별자본규제를활용한종속회사의요구자본환산치 + 2.비례성원칙을적용한종속회사의요구자본대응치 + 3.업권별자본규제를활용한관계회사의요구자본환산치`
- **허용오차**: 1 이하
- **참고**: 회사별로 해당 행이 존재하지 않을 수도 있음

### g. 지급여력비율 검증
- **규칙**: `다. 지급여력비율 : 가 ÷ 나 × 100 = 가.지급여력금액 / 나.지급여력기준금액(Ⅰ-Ⅱ+Ⅲ) × 100`
- **허용오차**: 1 이하

## 사용법

### 1. 환경 테스트
```bash
python run_validation.py
```

### 2. 전체 검증 실행
```bash
python disclosure_validation.py
```

### 3. 특정 회사만 검증
```python
from disclosure_validation import DisclosureValidator

validator = DisclosureValidator()
result = validator.validate_company("KR0001_메리츠화재해상보험")
```

## 출력 결과

### 콘솔 출력
- 각 회사별 검증 진행 상황
- 규칙별 통과/실패 상태
- 전체 요약 통계

### 엑셀 파일
- `disclosure_validation_results.xlsx`
  - **요약 시트**: 회사별 통과율 요약
  - **상세결과 시트**: 규칙별 상세 검증 결과

## 필요한 패키지

```bash
pip install pandas openpyxl
```

## 주요 특징

1. **스마트 행 검색**: 정확한 매칭이 실패하면 유사한 행명을 자동으로 찾습니다.
2. **유연한 오차 허용**: 1 이하의 오차는 정상으로 처리합니다.
3. **선택적 행 처리**: 일부 회사에만 존재하는 행들을 자동으로 처리합니다.
4. **상세한 로깅**: 각 단계별로 상세한 진행 상황을 출력합니다.
5. **엑셀 결과 저장**: 검증 결과를 엑셀 파일로 저장하여 후속 분석이 가능합니다.

## 오류 처리

- 파일 읽기 실패 시 해당 회사는 건너뛰고 계속 진행
- 특정 규칙 검증 실패 시 다른 규칙은 계속 검증
- 행을 찾을 수 없는 경우 유사한 행명을 제안

## 문제 해결

1. **파일을 찾을 수 없음**: `data\disclosure` 폴더 구조를 확인하세요.
2. **패키지 오류**: `pip install pandas openpyxl`을 실행하세요.
3. **행을 찾을 수 없음**: 엑셀 파일의 행명이 정확한지 확인하세요.
4. **계산 오차**: 허용오차(1)를 조정할 수 있습니다.
